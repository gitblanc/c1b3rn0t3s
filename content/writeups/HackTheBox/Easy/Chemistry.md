## Privilege Escalation

As I tried some common entry points and got nothing, I uploaded linpeas to the machine and inspected the results:

![](Pasted%20image%2020241227102128.png)

It seems that root is running something in the back inside `/opt/monitoring_site` called `app.py`, but I can't access to it. So, inspecting the ports I noticed port 8080 being used:

![](Pasted%20image%2020241227103651.png)

So I forwarded it to my machine:

```shell
# In my machine I'll run:
ssh -L 1010:127.0.0.1:8080 -N rosa@chemistry.htb
```

![](Pasted%20image%2020241227103752.png)

If I perform a `curl` request to it a Python server is being shown:

```shell
curl -I http://127.0.0.1:1010

[redacted]
Server: Python/3.9 aiohttp/3.9.1
```

So I searched for "*Python/3.9 aiohttp/3.9.1 cve*" and got this [cve article](https://ethicalhacking.uk/cve-2024-23334-aiohttps-directory-traversal-vulnerability/#gsc.tab=0)

I'll use the following payload:

```shell
curl -s --path-as-is "http://localhost:1010/assets/../../../../../etc/passwd"
```

![](Pasted%20image%2020241227105548.png)

> So there is a LFI and I can now read root flag :D
